Prasad Chalasani 1/7/2024 8:40‚ÄØPM
Hi all, We at Langroid have been fans of LanceDB for a while, and already had LanceDB as one of the vector-dbs (besides Qdrant and Chroma), since several months ago, but over the last few weeks, we made LanceDB the default vecdb in our DocChatAgent (our RAG-enabled agent). We also have a specialized LanceDocChatAgent which leverages the SQL filtering and direct Pandas dataframe ingestion (https://github.com/langroid/langroid/blob/main/langroid/agent/special/lance_doc_chat_agent.py). Here's something interesting we just rolled out last week: A LanceRAGTaskCreator which, given a LanceDocChatAgent, creates a 3-agent Computational RAG system with 2 additional agents:
  • A QueryPlannerAgent that, given a user's question like "What is the highest rated movie in the Crime category that is about prison escapes?, decides on a QueryPlan consisting of: (a) filter, (b) possibly rephrased query given the filter, (c) optional dataframe calculation string like df.loc[df['rating'].idxmax()] when the query might require a computation or aggregation
  • A QueryPlanCriticAgent that looks at the QueryPlan and Answer from the LanceDocChatAgent and gives feedback to the QueryPlanner on what to improve if anything. This system makes use of a couple of Tools (or function-calls) defined within Langroid: A QueryPlanTool and a QueryPlanFeedback tool.
This is a pretty unique way to showcase some of the strengths of LanceDB (native pandas interop, SQL filtering) and those of Langroid (elegant multi-agent task + tools orchestration). LanceRAGTaskCreator: https://github.com/langroid/langroid/blob/main/langroid/agent/special/lance_rag/lance_rag_task.py Test to see how it works -- https://github.com/langroid/langroid/blob/main/tests/main/test_lance_doc_chat_agent.py Two example scripts that use this 3-agent system:
  • movies dataset: https://github.com/langroid/langroid-examples/blob/main/examples/docqa/lance-rag-movies.py
  • github-issues: https://github.com/langroid/langroid-examples/blob/main/examples/docqa/lance-rag-gh-issues.py

Hey folks. I have been building something (going to OSS) for a couple of years that I think would align well with Lance. It is best described as a hierarchical, mutable, modular transformer-encoder architecture. So, basically you define a hierarchical schema (IE Customer -> Account -> Transactions -> [Timestamp, TrxnType, TrxnAmount, etc]), wherein each leaf node is of an arbitrary data type (category, number, datetime, raw text, hashable object, image, video, etc. - all which support optional / null values). The framework then creates a custom hierarchical transformer model that will take instances of this schema (all transactions observed under all accounts by all customers) and pretrain / finetune to solve arbitrary discriminative ML tasks (embedding, classification, regression, etc). 1. The framework is hierarchical, so you can also include monthly statements for each account and/or clickstream events, etc. 2. The framework is also mutable, such that you may add / remove / update nodes. 3. The framework in modular in that one can define new "plugins" to support new data types, and each data type comes with its own set of configurations (IE: categorical data types have a "max_vocab_size"-like parameter. The model architecture itself is a collection of very small transformer encoder blocks with custom pooling and a DAG-traversal-like forward pass. Altogether, the framework enables something interesting: users can train extremely high quality models with a mere ~10 million parameters that provide competitive performance to the modern alternative (XGBoost trained with a hundred handcrafted tabular features) but... in this case, no feature engineering is required. Just, train on your raw data directly from a datalake. Where does Lance fit in (I think?): well, I am having a hard time streaming media-based types (images/videos). That and, well, having pleasant, high quality models being integrated / versioned with the datasets upon which it was trained.
