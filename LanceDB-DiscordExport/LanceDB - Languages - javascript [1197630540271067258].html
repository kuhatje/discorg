WestonPace 6/18/2024 12:30â€¯PM
Would you consider this to be the best practice?
Yes, it's one approach. Keeping the old versions makes sense in some use cases (described more later) but it's perfectly valid to not do so.
If I run cleanupOldVersions(0), is there any risk of losing data?
No.
Is it standard practice to retain only the latest version? Does this last version file contain the complete dataset?
Yes, the latest version file contains the complete dataset. Old versions allow you to time travel / revert changes. If you don't need those features then it is a waste of space to keep the old versions.
Could you provide some insight into what the file manifest includes and its purpose?
The file manifest tells us what files are a part of your database. There's some more information on the manifest files in our description of the table format: https://lancedb.github.io/lance/format.html
for larger datasets, you would typically create batches of thousands~millions and run for each batch then save the entire batch
If you can do this then yes, batching is best. If you can't, then you can't. Whether it makes sense or not depends on the ingestion rate you need. If you need a really high ingestion rate then you'll need batching.
